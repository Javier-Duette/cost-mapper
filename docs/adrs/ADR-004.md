# ADR-004 — Flujo de trabajo IFC: importación, mapeo y sincronización

**Estado:** Aceptado  
**Pregunta original:** ¿Cómo hacemos que el modelo IFC generado por Revit sea 100% compatible? Propuesta inicial: plantilla + biblioteca de familias.

---

## Contexto

El vínculo entre el modelo 3D y el presupuesto es el núcleo funcional de Cost-Mapper. El IFC es un estándar abierto (ISO 16739) que cualquier software de modelado puede exportar, pero la calidad y el contenido de esa exportación varía enormemente. En el mercado paraguayo, la mayoría de los modelos no traerán clasificación NBR embebida. El sistema debe funcionar bien con cualquier IFC y mejorar progresivamente a medida que el usuario adopta mejores prácticas.

## Dos conceptos que no son lo mismo

**Biblioteca del proyecto** — ítems que el usuario preselecciona en Cost-Mapper antes de modelar. Es el universo de partidas candidatas para ese proyecto. De aquí sale el archivo de keynotes para Revit. Se puede copiar de un proyecto similar.

**Asignaciones del proyecto** — vínculos reales `GlobalId → ítem` generados durante el mapeo (automático por keynote o manual asistido). De aquí sale el presupuesto.

Un ítem puede estar en la biblioteca sin estar asignado. Un ítem puede asignarse aunque no estuviera en la biblioteca. Son dos tablas separadas en el modelo de datos.

## Decisión

### Flujo completo de trabajo

**FASE 1 — ANTES DE MODELAR (en Cost-Mapper):** Usuario crea proyecto → Construye la biblioteca del proyecto (busca ítems, copia de proyecto similar, o parte de plantilla por tipo de obra) → Configura y descarga el archivo de keynotes → Lo carga en Revit.

**FASE 2 — MODELADO (en Revit u otro software BIM):** Usuario aplica keynotes NBR a los elementos del modelo → Exporta el IFC con IfcClassificationReference.

**FASE 3 — IMPORTACIÓN (en Cost-Mapper):** Usuario importa el IFC → ifcopenshell extrae elementos 3D y calcula cantidades → Por cada elemento: si trae IfcClassificationReference → asignación automática; si no → mapeo asistido (sugerencia por IfcType, usuario acepta/ajusta) → Panel de revisión unificado → Presupuesto calculado automáticamente.

**El camino de keynotes es el preferido.** El mapeo asistido es el fallback para elementos sin clasificar.

### Herramientas IFC

- **Backend:** `ifcopenshell` (Python, open source, LGPL)
- **Frontend — visor 3D:** `@thatopen/components` (WebGL) — visualización bidireccional

### Keynotes NBR

Formato: texto plano, tres columnas separadas por tabulación (`CÓDIGO[TAB]DESCRIPCIÓN[TAB]CÓDIGO_PADRE`). Revit lo renderiza como árbol navegable.

Solo facetas con representación geométrica son aptas para keynotes:

| Faceta | Tipo | `bim_taggable` | Incluido por defecto |
| ------ | ---- | :---: | :---: |
| `3E` | Resultados de construcción | ✅ | ✅ activo |
| `4U` | Unidades / Espacios | ✅ | ✅ activo |
| `2C` | Componentes / Materiales | ✅ | ❌ inactivo (configurable) |
| `2N` | Funciones / Mano de obra | ❌ | ❌ nunca |
| `2Q` | Equipos de construcción | ❌ | ❌ nunca |
| `1S` | Servicios / Procesos | ❌ | ❌ nunca |

El archivo es generado por proyecto (desde la biblioteca del proyecto), no es global.

### Sincronización de cambios del modelo

**Las cantidades nunca se almacenan.** Cost-Mapper guarda `GlobalId → ítem`; la cantidad siempre se recalcula desde la geometría actual con `ifcopenshell` en cada importación.

| Tipo de cambio | Ejemplo | Acción del sistema |
| --- | --- | --- |
| **Cuantitativo** | El muro creció en área | Cantidad actualizada, presupuesto recalculado |
| **Cualitativo** | Espesor 15→20 cm | Alerta "verificar ítem", usuario confirma o reasigna |
| **Keynote cambiada** (fuente: `ifc_classification`) | Arquitecto cambió clasificación | Ítem actualizado automáticamente |
| **Keynote cambiada** (fuente: `user`) | Usuario corrigió manualmente | Alerta de conflicto, usuario decide |
| **Elemento nuevo** | GlobalId no existía | Pasa por flujo normal de mapeo |
| **Elemento eliminado** | GlobalId desaparecido | Alerta, usuario decide si elimina |

### MVP vs Post-MVP

- **MVP:** generación del keynote file desde la biblioteca del proyecto.
- **Post-MVP:** plantilla `.rte` de Revit y biblioteca de familias `.rfa`.

## Consecuencias

- El mapeo se almacena como `GlobalId → ítem`. Nunca por nombre, posición ni índice.
- El módulo de importación IFC soporta dos modos: **inicial** y **actualización** (diff).
- No rechaza archivos por falta de clasificación — reporta elementos sin asignar.
- Visor 3D y tabla de presupuesto están bidireccionalmente vinculados.
- Biblioteca y asignaciones son dos tablas separadas.
- No hay dependencia de Revit: cualquier software que exporte IFC válido funciona.
