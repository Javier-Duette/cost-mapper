## ADR-010 — Presupuesto manual pre-IFC mediante `manual_quantity` en `project_library`

**Estado:** Aceptado
**Pregunta original:** ¿Cómo permite el sistema construir un presupuesto útil antes de tener el módulo IFC (`ifc_importer`) implementado?

### Contexto

El modelo de datos oficial (`MODELO-DE-DATOS.md` sección 4) define `project_library` como una tabla de intención — los ítems que el usuario piensa usar en el proyecto — sin cantidades. El presupuesto real debía calcularse desde `project_assignments`, que depende de `ifc_elements`, que depende del pipeline IFC completo.

En la sesión de 2026-05-07, al implementar el módulo `budget/`, se constató que sin el pipeline IFC el módulo solo podría devolver una lista de ítems sin cantidades ni subtotales — un presupuesto que no puede calcular nada.

Se consideraron tres alternativas:

1. **No implementar budget/ ahora** — esperar al pipeline IFC completo. Descartado: el usuario necesita poder presupuestar manualmente desde el inicio.
2. **Tabla separada `budget_entries`** — nueva tabla con `project_id`, `item_id`, `manual_quantity`. Más limpio conceptualmente pero suma complejidad de schema sin resolver el problema raíz.
3. **Campo `manual_quantity` en `project_library`** — extiende la tabla existente con un campo nullable. Más simple y reversible: cuando el pipeline IFC esté listo, el módulo `budget/` puede leer las cantidades desde `project_assignments` en su lugar, y `manual_quantity` queda como fallback o se depreca.

### Decisión

Se agrega el campo `manual_quantity DECIMAL(14,4) NULL` a la tabla `project_library`. Es nullable: NULL significa "cantidad pendiente de asignar" (ya sea manualmente o desde IFC). El módulo `budget/` lee este campo para calcular subtotales.

El módulo `budget/` en MVP lee exclusivamente de `project_library JOIN catalog_items`. Cuando el módulo `ifc_importer` esté implementado, `budget/` se actualizará para leer de `project_assignments` cuando existan, con `manual_quantity` como fallback para ítems sin asignación IFC.

### Consecuencias

- `MODELO-DE-DATOS.md` sección 4 debe documentar `manual_quantity`.
- `ARQUITECTURA.md` sección 2.4 debe describir el comportamiento actual de `budget/` (MVP: lee de `project_library`) y el comportamiento futuro (post-IFC: lee de `project_assignments`).
- El campo `manual_quantity` **no es parte del estándar NBR 15965** ni del modelo de datos documentado por TCPO — es una extensión local del MVP.
- Cuando se implemente `ifc_importer`, revisar si `manual_quantity` debe mantenerse como entrada alternativa o deprecarse.
