## ADR-013 — `AGENTS.md` como fuente canonica para agentes IA

**Estado:** Aceptado
**Pregunta original:** ¿Debe `CLAUDE.md` seguir siendo el punto de entrada principal o debe existir una fuente neutral para todos los agentes?

### Contexto

Cost-Mapper ya es trabajado por distintos agentes y herramientas. Mantener reglas completas en `CLAUDE.md` hacia que el flujo operativo pareciera especifico de Claude Code, aunque tambien aplicaba a Codex, Copilot, Cursor y otros agentes.

La auditoria del 2026-05-08 detecto una bifurcacion documental: `AGENTS.md` aparecia como entrada neutral, mientras `CLAUDE.md` todavia competia como briefing completo. Esa duplicacion aumenta el riesgo de instrucciones divergentes.

### Decision

`AGENTS.md` queda como fuente canonica y neutral para todo agente IA del repositorio.

`CLAUDE.md` se conserva como archivo de compatibilidad/deprecacion controlada para herramientas que lo descubren automaticamente, pero no debe contener reglas propias ni decisiones nuevas. Su unica responsabilidad es redirigir a `AGENTS.md`.

Las skills internas del proyecto deben leer `AGENTS.md` como regla principal. Si se modifica el flujo de agentes, se actualizan en conjunto:

- `AGENTS.md`
- `.agents/skills/cost-mapper-agent/SKILL.md`
- `.agents/skills/cost-mapper-inspector/SKILL.md`, si afecta auditorias
- este ADR, solo si cambia la decision

### Consecuencias

- Se elimina la competencia entre dos briefings completos.
- `CLAUDE.md` deja de ser fuente de verdad y queda como puente estable.
- Los agentes que no son Claude Code ya tienen una entrada oficial y versionable.
- La documentacion y futuras auditorias deben comparar skills y reglas contra `AGENTS.md`, no contra `CLAUDE.md`.
